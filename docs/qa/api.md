По командному тестирования:

- **Парное** - 2 тестировщика или тестировщик + разработчик (могут быть разные комбинации) тестируют одно и то же.
- **Параллельное** - 2 тестировщика тестируют один и тото же модуль, но разные версии ПО (для поиска несоответствий)
- **A / B-тестирование** - исследование на ранних стадиях, часто используется в дизайне.  Допустим, есть версия дизайна А и версия дизайна B и необходимо понять какая версия лучше. Варианты показываются различным группам.
- **Gorilla/Monkey** - когда нужно протестировать какой-ниб сложный модуль и целая команда может его проверять (и тестировщики и разработчики).
- **Adhoc** (случайное, хаотичное) - что-то наподобие негативного тестирования, нестандартные сценарии. Свободное исследование.


Вид автоматизированного:

- **Мутационное** - вид автоматизированного тестирования, при котором в исходный код добавляются "мутации", какие-то случайные изменения, а потом код прогоняется тестами. Своего рода, проверка для авто-тестов. Если мутация остается незамеченной авто-тестами, хначит авто-тесты неэффективны.

Smoke обычно делают после билда. Результат либо да или нет. Если нет, то дальнейшее тестирование не проводится

New Feature Test - проверка только нового функционала. (часть модульного)
Retest - повторный тест после возвращения задачи на доработку.
